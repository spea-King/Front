# PRD: AI Interview Coach

실전 면접 감각을 훈련하는 AI 기반 면접 연습 서비스

---

## Intro & Goal

기업·직무 맥락을 반영한 질문을 음성으로 제공하고, 시간·구조 중심의 가이드를 통해 실전 면접 감각을 훈련하는 AI 기반 면접 연습 서비스.

기존 면접 준비 서비스들이 "답을 알려주는" 모델이라면, 우리 서비스는 **"말하는 연습을 돕는"** 훈련 도구이다. 노래방 메타포를 차용하여 답변 내용이 아닌 **리듬·구조·시간 감각**을 훈련시킨다.

**핵심 차별점:**
1. **Blind Audio Interview** — 질문 텍스트 없이 음성만 제공, 실제 면접 상황 재현
2. **Time & Structure Trainer** — 답변 시간 120초 고정 + 구조 가이드 (도입/핵심경험/결과)
3. **Pressure Mode** — 친절/압박 스타일 전환으로 실전 긴장감 재현
4. **Resume-Context Fit** — 자소서 + 기업/직무 맥락 기반 개인화 질문 생성
5. **No Score, Coaching** — 합불/점수 없이 훈련 피드백에 집중

---

## Who's It For?

- **취업 준비생** — 실전 면접 경험이 부족하여 말하기 연습이 필요한 구직자
- **이직 준비자** — 면접 감각을 되살리고 싶은 경력직
- **면접 긴장을 극복하고 싶은 사람** — 실제 면접 환경에서 시간 관리와 답변 구조화가 어려운 사람

---

## Why Build It?

- 기존 면접 준비 도구는 "정답"을 제공하지만, 실전에서는 **말하기 능력**이 핵심
- 면접 스터디 그룹은 시간/장소 제약이 크고, 일관된 피드백을 받기 어려움
- TTS/STT 기술 성숙으로 실전에 가까운 음성 면접 시뮬레이션이 가능
- 해커톤 프로젝트로 기술적 리스크 낮음 (검증된 API 활용)

## What Is It?

### Glossary -> 혼동할 수 있는 단어의 의미 구체적으로 정의! (누가 봐도 이해할 수 있게)

| 용어                  | 설명                                      |
| ------------------- | --------------------------------------- |
| **Session**         | 한 번의 면접 연습 세션 (5~10개 질문)                |
| **Question**        | AI가 생성한 면접 질문 (자소서 기반 / 직무 역량 / 상황 경험)  |
| **Answer**          | 사용자의 음성 답변 (STT로 텍스트 변환)                |
| **Structure Guide** | 답변 중 표시되는 구조 가이드 (도입 → 핵심 경험 → 결과/배운 점) |
| **Speed Signal**    | WPM 기준 말 속도 신호 (빠름/적정/느림)               |
| **Report**          | 면접 종료 후 제공되는 훈련 피드백 리포트                 |
| **Company Data**    | 기업 정보 JSON (인재상, 컬처핏, 직무 채용 포인트)        |

### User Types

| 유형 | 설명 |
|------|------|
| **Guest** | 랜딩 페이지만 볼 수 있는 비인증 사용자 |
| **User** | 면접 연습을 진행하는 일반 사용자 (인증 불필요, 세션 기반) |

- 모든 질문 답변 시간: 2분 고정

```
┌────────────────────────────────────────────┐
│           실전 면접 진행                     │
├────────────────────────────────────────────┤
│                                            │
│           [ 면접관 사진 ]                    │
│                                            │
│        🔊 질문 음성 재생 중                  │
│                                            │
│          남은 시간  01:35                    │
│   ██████████████████░░░░░░░░░░░░           │
│                                            │
│           구조 가이드                        │

└────────────────────────────────────────────┘
```

**Flow:**
1. AI 면접관이 TTS로 질문을 읽어줌
2. 사용자가 "답변 시작" 클릭 → 마이크 녹음 시작 + 프로그레스 바 카운트다운 (120초)
3. 답변 중 제공되는 보조 요소:
   - ⏱️ 남은 시간 프로그레스 바
   - 🎵 구조 가이드 (도입 → 핵심 경험 → 결과/배운 점) — 시간 경과에 따라 하이라이트 전환
   - 🚦 말 속도 신호 (WPM 기준 빠름/적정/느림)
4. 120초 경과 시 자동으로 다음 질문으로 전환
5. 마지막 질문 완료 시 면접 종료

**제공하지 않는 것:**
- ❌ 답변 내용 제안
- ❌ 문장 생성
- ⭕ 리듬·구조·시간만 보조 → "치트키가 아니라 훈련 도구"

```
│  - 내 답변: "..."                           │

└────────────────────────────────────────────┘
```

**리포트 구성:**

1. **전체 요약**
   - 3줄 요약 (좋았던 점 + 아쉬웠던 점)
   - 평균 답변 시간
   - 평균 말 속도 (텍스트 수 / 답변 시간으로 계산)
   - 질문별 시간 막대 그래프

2. **질문별 상세 분석**
   - 질문 텍스트
   - 실제 답변 시간 vs 권장 시간 (프로그레스 바)
   - 말 속도 (빠름/적정/느림)
   - 내 답변 (STT 텍스트)
   - AI 답변 제안 (GPT 생성)
   - 한 줄 평가 (자소서 맥락 반영, 강점/개선점)

**리포트에서 제공하지 않는 것:**
- ❌ 합격/불합격 판정
- ❌ 점수 매기기
- ⭕ 훈련 피드백에만 집중

## Functional Requirements

### REQ-001: Landing Page (Must)
- 서비스 소개 문구와 "면접 연습 시작" CTA 버튼 표시
- 버튼 클릭 시 기업/직무 선택 화면으로 이동

### REQ-002: 기업/직무 선택 (Must)
- TOSS 기업 카드 표시 (하드코딩 JSON)
- Frontend Developer 직무 선택 가능 (나머지는 비활성 표시)
- 선택 완료 후 "다음" 버튼으로 이력서 업로드 화면 이동

### REQ-003: 이력서/자소서 업로드 (Must)
- PDF 파일 업로드 기능
- 업로드된 파일에서 텍스트 추출 (Backend)
- "내용은 저장되지 않습니다" 안내 문구 표시
- 업로드 완료 후 "다음" 버튼으로 세션 설정 화면 이동

### REQ-004: 면접 세션 설정 (Must)
- 질문 개수 선택 (5~10개, 드롭다운)
- 면접 스타일 선택 (친절 / 압박)
- 면접관 목소리 선택 (남 / 여)
- "면접 시작" 버튼 클릭 시 질문 생성 API 호출 후 면접 화면 진입

### REQ-005: 면접 질문 생성 (Must)
- GPT-4o에 기업 데이터 + 자소서 텍스트 + 직무 정보 + 면접 스타일 전달
- 질문 유형: 자기소개(필수 첫 질문) + 자소서 기반 + 직무 역량 + 상황/경험
- 설정된 개수만큼 질문 생성
- 각 질문에 "왜 이 질문이 나왔는지" 맥락 정보 포함 (리포트용)

### REQ-006: TTS 질문 음성 재생 (Must)
- OpenAI TTS API로 면접관 음성 생성
- 남/여 목소리 설정 반영
- 친절/압박 스타일에 따라 프롬프트 톤 조절
- 질문 음성 재생 완료 후 답변 시작 가능

### REQ-007: 음성 답변 녹음 + STT (Must)
- 브라우저 MediaRecorder API로 마이크 녹음
- 답변 완료 시 녹음 파일을 Backend로 전송
- OpenAI Whisper API로 음성→텍스트 변환
- 변환된 텍스트와 답변 시간(초) 저장

### REQ-008: 답변 타이머 + 자동 전환 (Must)
- 120초 카운트다운 프로그레스 바
- 남은 시간 숫자 표시 (MM:SS)
- 120초 경과 시 자동으로 녹음 중지 + 다음 질문으로 전환
- 사용자가 조기 종료도 가능 ("다음 질문" 버튼)

### REQ-009: 구조 가이드 표시 (Should)
- 답변 시간에 따라 3단계 구조 하이라이트: 도입(0~30초) → 핵심 경험(30~90초) → 결과/배운 점(90~120초)
- 현재 단계를 시각적으로 강조 (색상 변경 또는 하이라이트)

### REQ-010: 말 속도 신호 (Should)
- STT 실시간 또는 답변 완료 후 WPM 계산
- 빠름(>180 WPM) / 적정(120~180 WPM) / 느림(<120 WPM) 신호등 표시

### REQ-011: 면접 종료 + 로딩 화면 (Must)
- 마지막 질문 완료 시 "면접이 종료되었습니다" 화면
- "분석 중..." 로딩 애니메이션
- "결과 보기" 버튼 클릭 시 리포트 생성 API 호출

### REQ-012: 리포트 생성 (Must)
- GPT-4o에 질문 목록 + 각 답변 텍스트 + 답변 시간 + 기업/직무 맥락 전달
- 전체 요약 (3줄 요약 + 평균 답변 시간 + 평균 말 속도)
- 질문별 상세 분석 (내 답변 + AI 답변 제안 + 한 줄 평가)

### REQ-013: 리포트 시각화 (Must)
- 질문별 답변 시간 막대 그래프
- 각 질문의 시간/속도 인디케이터
- 질문별 상세 분석 카드 UI

## Non-Functional Requirements

### NFR-001: 응답 시간
- TTS 음성 생성: < 5초
- STT 텍스트 변환: < 10초 (2분 녹음 기준)
- 질문 생성 API: < 15초
- 리포트 생성 API: < 20초

### NFR-002: 브라우저 호환성
- Chrome 최신 버전 지원 (MediaRecorder API, Web Audio API)
- 마이크 권한 요청 처리

### NFR-003: 개인정보 보호
- 업로드된 PDF 파일은 세션 종료 시 서버에서 삭제
- 음성 녹음 파일은 STT 변환 후 삭제
- 사용자 개인정보 영구 저장하지 않음

## Data Models

### Session
```
{
  session_id: ObjectId,
  company_id: string,        // "toss"
  job_id: string,            // "frontend"
  resume_text: string,       // 추출된 자소서 텍스트
  question_count: number,    // 5~10
  interview_style: string,   // "friendly" | "pressure"
  voice_gender: string,      // "male" | "female"
  status: string,            // "setup" | "in_progress" | "completed"
  created_at: datetime
}
```

### Question
```
{
  question_id: ObjectId,
  session_id: ObjectId,
  order: number,             // 질문 순서 (1부터)
  question_text: string,     // 질문 텍스트
  question_type: string,     // "intro" | "resume" | "competency" | "situational"
  context: string,           // 질문 출제 맥락 (왜 이 질문이 나왔는지)
  tts_audio_url: string,     // TTS 생성된 음성 파일 경로
  answer_text: string,       // STT 변환된 답변 텍스트
  answer_duration: number,   // 답변 시간 (초)
  answer_wpm: number,        // 분당 단어 수 (말 속도)
  created_at: datetime
}
```

### Report
```
{
  report_id: ObjectId,
  session_id: ObjectId,
  summary: string,           // 3줄 요약
  avg_answer_time: number,   // 평균 답변 시간 (초)
  avg_wpm: number,           // 평균 말 속도
  question_analyses: [
    {
      question_id: ObjectId,
      answer_time: number,
      speed_label: string,   // "fast" | "normal" | "slow"
      ai_suggested_answer: string,
      one_line_review: string
    }
  ],
  created_at: datetime
}
```

## Out of Scope (MVP에서 제외)

- 사용자 회원가입 / 로그인 / 인증
- 면접 이력 저장 / 다시 보기
- 토스 외 다른 기업 지원
- Frontend 외 다른 직무 지원
- 실시간 STT (답변 중 텍스트 표시)
- 면접관 영상/애니메이션 (정적 이미지만 사용)
- 모바일 최적화
- 배포 (로컬 개발 환경에서만 동작)
- 다국어 지원

## Open Questions & Risks

| # | Question / Risk | Impact | Mitigation |
|---|----------------|--------|------------|
| 1 | OpenAI API 비용이 해커톤 예산을 초과할 수 있음 | High | API 호출 최소화, 질문 캐싱 고려 |
| 2 | TTS 음성 생성 지연이 면접 UX를 해칠 수 있음 | Medium | 모든 질문 TTS를 면접 시작 전에 일괄 생성 |
| 3 | Whisper STT 한국어 정확도가 낮을 수 있음 | Medium | 한국어 프롬프트 최적화, 발화 길이 제한 |
| 4 | 브라우저 마이크 권한 거부 시 서비스 이용 불가 | High | 권한 요청 UX 가이드 제공 |
| 5 | 2분 녹음 파일 크기가 클 수 있음 | Low | WebM/Opus 코덱 사용으로 압축 |

## Success Metrics (훈련 도구 관점)

- 시연 시 전체 플로우가 끊김 없이 동작
- TTS 질문 → 음성 답변 → 리포트 생성까지 3분 이내 완료 (5개 질문 기준)
- 리포트에서 질문별 답변 시간/속도/AI 피드백이 정상 표시
- "면접 연습 서비스"라는 컨셉이 시연만으로 즉시 전달됨
